## Recently fixed (this pass)

- **[FIXED] SQL importer silently drops tables with paste-style errors** — common copy-paste bugs caused tables like `package_menus` (missing comma before a table-level `PRIMARY KEY`) and `role_menus` (inline column PK + table-level composite PK) to fail with `near "(": syntax error` / `more than one primary key`, but the failures were swallowed and only showed in the toast count. Two fixes:
  1. `lib/sql-import.ts` `autoRepairCreateTableBodies` — paren-balanced scan over each `CREATE TABLE ( ... )` body that (a) inserts a comma before any `PRIMARY KEY|UNIQUE|FOREIGN KEY|CHECK|CONSTRAINT` clause that follows a column-def without one, and (b) strips a redundant inline `PRIMARY KEY` when the table also has a composite table-level `PRIMARY KEY (...)` on a different column. Verified end-to-end: the user-provided 5-table schema now parses 5/5 instead of 3/5.
  2. New `parseSqlToSchemaDetailed` returns `{ schema, warnings, dialect }`. `sql-preview.tsx` shows a per-statement warning list under the editor (with the offending excerpt) when `warnings.length > 0` and the toast becomes a yellow "N statements skipped" instead of a green success. The legacy `parseSqlToSchema` keeps the old signature so other callers don't need to change.
- **[FIXED] SQL Import editor was a transparent-textarea overlay hack** — caret/highlight could drift on long lines, no line numbers, no proper undo, no bracket matching, awkward to add or remove lines. Replaced with a real `CodeMirror 6` editor (`components/sql-panel/sql-code-editor.tsx`):
  - Per-dialect SQL grammar (PostgreSQL / MySQL / SQLite / MSSQL / generic), `upperCaseKeywords`.
  - Line numbers + active-line highlight, fold gutter.
  - Bracket matching + auto-close pairs.
  - Multi-cursor (Alt+click), search (Ctrl+F), undo/redo (Ctrl+Z / Ctrl+Y), keyword autocomplete (Ctrl+Space), rectangular selection.
  - `EditorView.lineWrapping`; tab indents 2 spaces.
  - Theme follows `next-themes` resolved theme (`oneDark` for dark, `light` for light) without an SSR/hydration mismatch (`useSyncExternalStore`-based `useIsClient` hook avoids the previous setState-in-effect lint error).
- Installed deps: `@uiw/react-codemirror`, `@codemirror/lang-sql`, `@uiw/codemirror-theme-vscode` (`@codemirror/theme-one-dark` was tried first then dropped — see VSCode-theme entry below).
- **[FIXED] Editor uses VSCode dark by default.** Switched the SQL editor's theme from `oneDark` to `@uiw/codemirror-theme-vscode`'s `vscodeDark` / `vscodeLight` so the dark variant matches the deeper VSCode default rather than One Dark Pro. Removed the now-unused `@codemirror/theme-one-dark` dependency. Trimmed dead code: `useIsClient` SSR gate is gone (the project's `useTheme` is hydration-safe), placeholder light-fallback string is gone (we always pass a real theme object now), and `extensions` is collapsed to a single line.
- **[FIXED] Editor background matches the surrounding panel.** `components/sql-panel/sql-code-editor.tsx`. VSCode's own `#1e1e1e` looked lighter than this app's deep-black panel and the editor stood out as a grey rectangle against the dark UI. Added a `surfaceTheme` `EditorView.theme(...)` extension layered after the base `vscodeDark` theme so the wrapper, `.cm-scroller`, and `.cm-gutters` all use `var(--color-background)` and `var(--color-foreground)`. Active-line stripe + active-line gutter use a 6% foreground tint via `color-mix(in oklab, …)`; selection uses 30% primary tint; gutter border uses `var(--color-border)`. The override is `useMemo`'d so React doesn't recreate the extension on every render. Same overrides apply in light mode against the light theme tokens — the editor inherits the panel's background no matter which theme is active.
- **[FIXED] CodeMirror editor wasn't following app theme.** `components/sql-panel/sql-code-editor.tsx`. Was importing `useTheme` from `next-themes`, but this app's provider is the in-house `components/theme-provider`. The next-themes hook returned a stale resolvedTheme so the editor stayed pinned on the light theme even when the app was dark. Switched to the project's `useTheme`. Dropped the no-longer-needed `useIsClient` SSR gate; `resolvedTheme` from the project provider is hydration-safe on its own.
- **[FIXED] Editor pushed footer buttons out of view as user typed.** `components/sql-panel/sql-preview.tsx`. The Import tab used `flex-1` on the editor row but no `min-h-0`, so CodeMirror's growing content stretched the column past the viewport — the Clear / Import to Canvas buttons disappeared off the bottom. CodeMirror's wrapper also got `h-full min-h-0` so the internal scrollbar takes over instead of pushing siblings around.

Editor itself grows / shrinks with the wrapper at any breakpoint via `width: 100%`.

## Critical

- **[FIXED] No timeout on LLM invocations.** `lib/langchain/ai.ts`. Added a 45 s wall-clock cap via three layers: provider-level `timeout` constructor option (OpenAI / OpenRouter / Grok / Ollama / Anthropic via `clientOptions.timeout`), `AbortSignal.timeout` on every `chain.invoke(...)`, and a final `Promise.race` against a hard deadline so providers that swallow the signal (some structured-output adapters do) still can't hang a route. Applies to `generateSchema`, `explainSchema`, `fixSchema`, `generateQuery`, `optimizeQuery`, `explainQuery`, `documentSchema`, `adviseIndexes`.
- **[FIXED] No timeout on database introspection.** `app/api/introspect/route.ts`. Wrapped `introspectDatabase(...)` in a `Promise.race` against a 25 s deadline (>= driver-level 8 s connect + 15 s query). On timeout the route now returns `504` with a clear error; other introspection failures continue to return `502`.
- **[FIXED] Prompt injection unchecked in AI routes.** Two-layer defence:
  1. `app/api/ai/route.ts` validates the body with a strict Zod schema (`action` must be one of a fixed enum, each text field is bounded — 8k chars for free text, 200k chars for schema JSON). Each handler now explicitly enforces the fields it needs and returns 400 on missing/empty input. Replaces the previous `body as { action; payload }` cast.
  2. `lib/langchain/ai.ts` exports `sanitizeUserInput()` which is applied to every user-controlled field before interpolation: clamps length, strips ChatML / Anthropic / OpenAI role-control tokens (`<|system|>`, `<|im_start|>`, `</user>`, …), and replaces obvious jailbreak preambles ("ignore previous instructions", `### system:`, `assistant:` at start-of-line) with `[redacted-instruction]`. Doesn't claim to defeat a determined adversary — that's the model's job — but removes the easy footguns.

## High

- **[FIXED] Stale `activeSchemaId` in autosave.** `lib/workspace-context.tsx`. Hardened the debounce timer body: the schema id is snapshotted into `targetSchemaId` at schedule time and passed explicitly to `performSave(...)`. Inside the timer we re-check `activeSchemaIdRef.current === targetSchemaId` so a switch-then-edit-then-fire sequence bails. Also switched from the closure-captured `schema` to `schemaRef.current` so a synchronous mutation in the same microtask the timer is dispatched in still saves the freshest state, and added a provider-unmount cleanup that clears `saveTimerRef` to stop a stale timer firing against a torn-down setter under React strict-mode.
- **[FIXED] `importAiSchema` doesn't validate relation endpoints.** `lib/schema-store.tsx`. The basic `null`-filter for unresolved endpoints was already in place; tightened the import path further: dedupe table names and column-name-within-table (drop later duplicates instead of silently overwriting in the lookup map), wire each surviving relation's source column with a populated `references = { table, column }` plus a `REFERENCES` constraint, and strip stale `REFERENCES` constraints from columns whose declared FK didn't survive the validity filter so the column doesn't claim to be a FK with no edge to back it up.
- **[FIXED] Removing a column doesn't repair indexes / relations referencing it.** `lib/schema-store.tsx`. `removeColumn` already dropped relations on either side and trimmed indexes within the deleted column's table — verified. The remaining gap was `removeTable` / `removeTables`: deleting a table dropped its relations but left columns on *other* tables with stale `references = { table: <deleted-name>, ... }` and a dangling `REFERENCES` constraint. Both now strip those references on surviving tables (singular and bulk variants), matched by deleted-table name.
- **[FIXED] Comparison API casts `schemaJson` without validation.** Added `lib/schema-validate.ts` exposing a Zod-backed `parseSchemaJson(input): { ok, schema } | { ok:false, error }`. Wired into `app/api/schemas/compare/route.ts` (both `left.schemaJson` and `right.schemaJson` are validated; route returns `422` with which side is malformed and the failing path) and into `app/api/reviews/[id]/route.ts` (see next entry). Replaces the previous `as unknown as Schema` casts.
- **[FIXED] Review-approval merges proposed JSON without structural check.** `app/api/reviews/[id]/route.ts`. `proposedJson` is now `parseSchemaJson`-validated *before* the transaction starts. On shape mismatch the route returns `422` with `"Proposed schema is malformed: <field>"` and the live row is untouched. Approve / reject / decided-status guards remain in place.
- **[FIXED] `use-stored-state` setState-in-effect + validate-after-set.** `lib/use-stored-state.ts` rewritten on top of `useSyncExternalStore` (the React-recommended hydration-safe pattern):
  - Server snapshot is `initial`; client first-paint snapshot is also `initial`; persisted value commits on the next render after mount, so no setState-in-effect.
  - `validate(parsed)` runs *before* the parsed value is ever returned as state — corrupt JSON falls back to `initial` without flashing bad data through React.
  - Snapshots are cached per-key so `getSnapshot` returns referentially-stable values (avoids `useSyncExternalStore`'s "infinite loop" warning).
  - Cross-tab sync via `storage` event invalidates the cache and re-emits.
  - Same-tab sync via a manual pub/sub: a setter in tab A notifies any other consumer of the same key in tab A.
- **[FIXED] `syncNodesFromSchema` uses `selectedTableId` from closure.** `components/canvas/schema-canvas.tsx`. The sync now reads `selectedTableIdRef.current` inside its body, so a schema mutation that doesn't change selection stops carrying a stale `selected` flag. `selectedTableId` was removed from the callback's dep array (no longer needed) and a separate small effect kicks the sync when the selection id flips so the visual highlight updates without waiting for the next schema change.
- **[FIXED] Concurrent schema PATCH has no optimistic-concurrency check.** `app/api/schemas/[id]/route.ts` + `lib/workspace-context.tsx`. Wire diagram:
  1. The route now accepts an optional `expectedVersion: number` in the PATCH body. When present it uses `prisma.schema.updateMany({ where: { id, version: expectedVersion }, data })` — a single conditional write that closes the read-then-write window. If 0 rows match, the route returns `409 VERSION_MISMATCH` with `{ currentVersion, updatedAt }` so the client can resync.
  2. `loadSchemaIntoCanvas` snapshots the server-observed `version` into `currentVersionRef`. Every PATCH (autosave + Ctrl+S `saveNow`) now sends `expectedVersion: currentVersionRef.current`. The successful response carries the new `version`, which we re-snapshot for the next call.
  3. On 409 the client toasts "schema was updated in another tab" and calls `loadSchemaIntoCanvas(id)` to realign with the server. The skip-next-autosave guard prevents an immediate re-PATCH that would clobber the freshly-loaded data.
  4. Omitting `expectedVersion` (older clients) preserves the legacy last-write-wins behaviour rather than breaking those sessions.
- **[FIXED] `saveTimerRef` not cleared on unmount.** `lib/workspace-context.tsx`. Provider-unmount `useEffect` cleanup now: clears `saveTimerRef`, sets an `unmountedRef` so any post-mount async branches bail before touching state, and aborts every in-flight PATCH via a per-call `AbortController` registered in `inFlightAbortersRef`. The save handler treats `AbortError` as a silent skip (no error toast, no setState) so dev strict-mode and production route changes both unmount cleanly without firing setState on a torn-down tree or surfacing spurious "Save failed" toasts.

## Medium

- **[FIXED] `updateRelation` accepts arbitrary endpoints / types.** `lib/schema-store.tsx`. Now narrows `updates.type` against the literal enum and resolves the candidate (incoming-or-current) `sourceTable` / `sourceColumn` / `targetTable` / `targetColumn` against `prev.tables` before applying. Bails (returns `prev`) on bad type or unresolved endpoints — orphan relations can no longer be persisted by an external caller.
- **[FIXED] `updateColumn` rename doesn't update inbound `references`.** `lib/schema-store.tsx`. On a rename (column name actually changes, no in-table collision) the reducer also walks *every other table* and rewrites any column whose `references.table === oldTable.name && references.column === oldCol.name` so its `references.column` points at the new name. Table-name guard prevents collisions with same-named columns in unrelated tables.
- **[FIXED] `updateTableName` blind spot for `references.table`.** `lib/schema-store.tsx`. Same family of fix: on a real rename, walks peers and rewrites every column whose `references.table === oldName` to the new name. Now table-rename + column-rename together keep `references` strings consistent so the SQL generator stops emitting broken FKs.
- **[FIXED] No `maxTokens` on LLM calls.** `lib/langchain/ai.ts`. Added `LLM_MAX_TOKENS = 4096` and pass it through every provider constructor: OpenAI / OpenRouter / Grok / Ollama / Mistral / Anthropic / Bedrock as `maxTokens`, Google as `maxOutputTokens`. Caps any single completion at ~4 KB tokens so a 200-table schema doc-gen can't bill the user's key for a multi-MB response.
- **[FIXED] `canUndo` / `canRedo` recompute on every render.** `lib/schema-store.tsx`. Replaced the ref-during-render reads (which also triggered the `react-hooks/refs` lint error) with two real `useState` flags driven by a `syncHistoryFlags()` call inside every history mutation. State setters use the prev-vs-next equality form so no spurious render fires when the value didn't actually flip. Removed the now-unused `useReducer` import + `bumpHistory` tick counter.
- **[FIXED] Fragile `onSelectionChange` compare.** `components/canvas/schema-canvas.tsx`. Replaced the index-by-index loop with an order-independent compare: `[...prev].sort().join("")` vs `[...next].sort().join("")`. Same contents in any order are now treated as equal so RF's render-driven re-fires don't ping-pong selection state.
- **[FIXED] `autoArrange` clobbered the saved viewport.** `components/canvas/schema-canvas.tsx`. After the `fitView` animation settles (`setTimeout(saveViewport, 550)`), the new arrangement's viewport is persisted. Switching schema tabs and coming back now lands on the freshly-arranged view instead of the pre-arrange one.
- **[FIXED] No CSRF protection on mutating API routes.** Added `lib/csrf.ts` exporting `requireJsonContentType(req)` which 415s any POST/PATCH/PUT/DELETE that isn't `application/json` (DELETE without a body is allowed). HTML `<form>` submissions can only emit `application/x-www-form-urlencoded` / `multipart/form-data` / `text/plain`, so this blocks the residual same-site cross-page form attack that SameSite=Lax cookies don't cover. Wired into every mutating route: `/api/schemas/[id]`, `/api/schemas/compare`, `/api/schemas/[id]/duplicate`, `/api/schemas/[id]/share`, `/api/schemas/[id]/reviews`, `/api/schemas/[id]/versions`, `/api/schemas/[id]/versions/[versionId]`, `/api/projects`, `/api/projects/[id]`, `/api/projects/[id]/schemas`, `/api/workspaces`, `/api/workspaces/[id]`, `/api/profile`, `/api/settings`, `/api/settings/validate-key`, `/api/ai`, `/api/introspect`, `/api/reviews/[id]`, `/api/mock-db`, `/api/trash/projects/[id]`, `/api/trash/schemas/[id]`. NextAuth's own endpoints are deliberately not touched (its CSRF flow is independent).
- **[VERIFIED] AI rate-limit per-user.** `app/api/ai/route.ts`. Confirmed: rate limit is keyed on `ai:${session.user.id}` — per-user, not per-workspace. The remaining cross-user concern is the user's own *provider* quota when their key is used in a multi-tab session, but that's expected behaviour because it's their own key.
- **[FIXED] Mock-DB execution: no timeout / unbounded dataset.** `lib/mock-db.ts`. sql.js runs synchronously on the main thread so a true `setTimeout` kill is impossible without moving to a Web Worker; until that refactor lands we enforce input bounds that prevent the runaway in the first place: `MAX_ROWS_PER_TABLE = 500`, `MAX_TOTAL_INSERT_ROWS = 5000` (`tableCount × rowsPerTable`), `MAX_QUERY_LEN = 10_000`. Each is rejected up-front with a clear error. Post-execution, queries that took >5 s log a `console.warn` so slow queries are at least visible.
- **[FIXED] `schema-canvas` `onMoveEnd` localStorage spam.** `components/canvas/schema-canvas.tsx`. `saveViewport` now debounces by 250 ms via a `viewportSaveTimerRef`, so a continuous trackpad pan writes at most ~4 times per second instead of dozens. Provider-unmount effect clears the debounce timer to keep storage writes from firing against a torn-down tree.
- **[FIXED] `generateMockData` orphan FK rows.** `lib/mock-db.ts`. `runMockQuery` now tracks the *actual* number of rows successfully inserted into each parent table (`parentRowCounts: Map<tableId, count>`) and `generateInsertStatements` clamps each FK column's `maxId` to that count. Empty-parent edge case (cyclic graph or all-failed inserts) emits `NULL` instead of pointing at row id 0. Combined with the existing `topologicalSort` parent-first ordering, child FK values are now always drawn from the actual valid parent id range.
- **[FIXED] `bulk-export` ZIP lacks a top-level folder.** `lib/bulk-export.ts`. Every entry name now starts with `${base}/` so extracting on Windows / macOS produces a single folder instead of scattering 12 loose files into the user's working directory.

## Low

- **`isValidConnection` only blocks self-loops.** `components/canvas/schema-canvas.tsx`. Doesn't prevent connecting a column to itself across tables (cross-type FK from `text` → `int`). **Fix:** check column type compatibility and block when types are incompatible (warn / require user override).
- **`introspectMssql` uses `Encrypt=true; TrustServerCertificate=true` defaults.** `lib/db-introspect.ts`. Fine for dev but should warn when both are on in production. **Fix:** show a yellow note in `connect-db-dialog` when the connection string omits encryption flags.
- **`splitSqlStatements` doesn't handle MySQL `DELIMITER //` blocks.** `lib/sql-import.ts`. mysqldump output that uses `DELIMITER //` for stored procedures will mis-split. **Fix:** track current delimiter; switch when `DELIMITER` appears at line start.
- **`detectDialect` looks for backticks last.** `lib/sql-import.ts`. A dump that includes both PG-style `TIMESTAMPTZ` (in a comment string) and MySQL backticks is mis-classified PG. Edge case but worth a smarter score-based detector. **Fix:** count signal hits per dialect and pick the max.
- **`schema-lint` doesn't check for cycles.** `lib/schema-lint.ts`. A → B → A FK cycle is valid SQL but commonly a design smell. **Fix:** add a `cycle-detected` warning.
- **Empty-catch on `sign-up` JSON parse.** `app/api/sign-up/route.ts:47`. `.catch(() => ({}))` makes "invalid JSON" indistinguishable from "valid but missing fields". **Fix:** return 400 with `"Invalid JSON"` on the catch path.
- **`console.error` in API routes can leak server traces.** `app/api/sign-up/route.ts:99,108` and similar. In serverless logs that's fine; if anyone pipes server logs to a client telemetry endpoint, full stacks leak. **Fix:** log a redacted summary plus a request id; keep stacks server-side only.
- **`useStoredState` doesn't sync across tabs.** `lib/use-stored-state.ts`. Two tabs editing the same schema show drifted UI for stored state (panel layout, tab strip). **Fix:** subscribe to `storage` events.
- **`relation-type-dialog` displays column names as plain strings, no escaping.** `components/canvas/relation-type-dialog.tsx`. If a column name contains HTML characters they'd render as text — fine in React but if anything ever does `dangerouslySetInnerHTML` on those names downstream, it'd break. **Fix:** keep using JSX text nodes (already safe); document.
- **`exportPng` doesn't strip controls/UI overlays.** `lib/export-utils.ts`. The exported image includes whatever UI is on top of the canvas at click time. **Fix:** hide controls during snapshot, restore after.
- **Memory leak in real-time presence.** `lib/use-collab.ts` (presence broadcast). Peer disconnect events not always cleaned from `Y.Awareness` if the WebRTC channel drops mid-handshake. **Fix:** add a 30 s "stale-peer" sweep on the local awareness map.
- **`schema-templates` JSON parsed without validation.** Templates ship from `lib/schema-templates.ts`; any third-party-added template can crash the canvas. **Fix:** Zod-validate before applying.
- **`use-relation-form` defaults pick first non-PK column.** `lib/hooks/use-relation-form.ts`. If a table has only PKs, it falls back to `columns[0]`, which may be the PK itself, creating a self-FK. **Fix:** reject the relation creation when no non-PK column exists.
- **Accessibility: drag handles missing `aria-label`.** `components/sidebar/schema-sidebar.tsx`. Screen-reader users can't tell which row's drag handle is focused. **Fix:** add `aria-label={`Drag ${tableName}`}`.
- **No tooltip on the new SQL Server tab in connect dialog.** `components/workspace/connect-db-dialog.tsx`. The placeholder mentions both URL and ADO formats; a hover tip would help discoverability.
- **Rate limit uses an in-memory map.** `lib/rate-limit.ts`. Reset between Lambda cold starts; effectively no limit on serverless. **Fix:** back with Redis when running multi-instance.
- **`schema-store` `replaceSchema` skips push to history.** Switching schemas, importing, and AI-generation all blow away history. Documented but surprising — undo doesn't take you back to the previous schema. **Fix:** push a "boundary" marker that's labeled distinctly in the UI.

## Notes

- The recent SQL-import overhaul (PG `$$ … $$` aware splitter, statement allow-list, balanced-paren `GENERATED ALWAYS AS` strip, partition handling) is covered by an end-to-end probe in the working tree but not committed to the test suite. Adding a vitest that exercises `parseSqlToSchema` against a few public-domain DDL fixtures would catch regressions cheaply — currently no `sql-import.test.ts` exists.
- `tsc --noEmit` passes; ESLint has 19 errors / 23 warnings, all pre-existing in `lib/schema-store.tsx` and `lib/use-stored-state.ts` (refs accessed during render, setState in effect). Worth a focused cleanup pass — these are real React 18+ correctness issues, not just style.
- No request-id / structured logging anywhere; debugging a stuck introspect or AI call in production means correlating wall-clock guesses. A 5-line `nanoid()` request id middleware in each API route would pay for itself the first incident.
