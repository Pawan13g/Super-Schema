## Critical

- **[FIXED] No timeout on LLM invocations.** `lib/langchain/ai.ts`. Added a 45 s wall-clock cap via three layers: provider-level `timeout` constructor option (OpenAI / OpenRouter / Grok / Ollama / Anthropic via `clientOptions.timeout`), `AbortSignal.timeout` on every `chain.invoke(...)`, and a final `Promise.race` against a hard deadline so providers that swallow the signal (some structured-output adapters do) still can't hang a route. Applies to `generateSchema`, `explainSchema`, `fixSchema`, `generateQuery`, `optimizeQuery`, `explainQuery`, `documentSchema`, `adviseIndexes`.
- **[FIXED] No timeout on database introspection.** `app/api/introspect/route.ts`. Wrapped `introspectDatabase(...)` in a `Promise.race` against a 25 s deadline (>= driver-level 8 s connect + 15 s query). On timeout the route now returns `504` with a clear error; other introspection failures continue to return `502`.
- **[FIXED] Prompt injection unchecked in AI routes.** Two-layer defence:
  1. `app/api/ai/route.ts` validates the body with a strict Zod schema (`action` must be one of a fixed enum, each text field is bounded — 8k chars for free text, 200k chars for schema JSON). Each handler now explicitly enforces the fields it needs and returns 400 on missing/empty input. Replaces the previous `body as { action; payload }` cast.
  2. `lib/langchain/ai.ts` exports `sanitizeUserInput()` which is applied to every user-controlled field before interpolation: clamps length, strips ChatML / Anthropic / OpenAI role-control tokens (`<|system|>`, `<|im_start|>`, `</user>`, …), and replaces obvious jailbreak preambles ("ignore previous instructions", `### system:`, `assistant:` at start-of-line) with `[redacted-instruction]`. Doesn't claim to defeat a determined adversary — that's the model's job — but removes the easy footguns.

## High

- **[FIXED] Stale `activeSchemaId` in autosave.** `lib/workspace-context.tsx`. Hardened the debounce timer body: the schema id is snapshotted into `targetSchemaId` at schedule time and passed explicitly to `performSave(...)`. Inside the timer we re-check `activeSchemaIdRef.current === targetSchemaId` so a switch-then-edit-then-fire sequence bails. Also switched from the closure-captured `schema` to `schemaRef.current` so a synchronous mutation in the same microtask the timer is dispatched in still saves the freshest state, and added a provider-unmount cleanup that clears `saveTimerRef` to stop a stale timer firing against a torn-down setter under React strict-mode.
- **[FIXED] `importAiSchema` doesn't validate relation endpoints.** `lib/schema-store.tsx`. The basic `null`-filter for unresolved endpoints was already in place; tightened the import path further: dedupe table names and column-name-within-table (drop later duplicates instead of silently overwriting in the lookup map), wire each surviving relation's source column with a populated `references = { table, column }` plus a `REFERENCES` constraint, and strip stale `REFERENCES` constraints from columns whose declared FK didn't survive the validity filter so the column doesn't claim to be a FK with no edge to back it up.
- **[FIXED] Removing a column doesn't repair indexes / relations referencing it.** `lib/schema-store.tsx`. `removeColumn` already dropped relations on either side and trimmed indexes within the deleted column's table — verified. The remaining gap was `removeTable` / `removeTables`: deleting a table dropped its relations but left columns on *other* tables with stale `references = { table: <deleted-name>, ... }` and a dangling `REFERENCES` constraint. Both now strip those references on surviving tables (singular and bulk variants), matched by deleted-table name.
- **[FIXED] Comparison API casts `schemaJson` without validation.** Added `lib/schema-validate.ts` exposing a Zod-backed `parseSchemaJson(input): { ok, schema } | { ok:false, error }`. Wired into `app/api/schemas/compare/route.ts` (both `left.schemaJson` and `right.schemaJson` are validated; route returns `422` with which side is malformed and the failing path) and into `app/api/reviews/[id]/route.ts` (see next entry). Replaces the previous `as unknown as Schema` casts.
- **[FIXED] Review-approval merges proposed JSON without structural check.** `app/api/reviews/[id]/route.ts`. `proposedJson` is now `parseSchemaJson`-validated *before* the transaction starts. On shape mismatch the route returns `422` with `"Proposed schema is malformed: <field>"` and the live row is untouched. Approve / reject / decided-status guards remain in place.
- **[FIXED] `use-stored-state` setState-in-effect + validate-after-set.** `lib/use-stored-state.ts` rewritten on top of `useSyncExternalStore` (the React-recommended hydration-safe pattern):
  - Server snapshot is `initial`; client first-paint snapshot is also `initial`; persisted value commits on the next render after mount, so no setState-in-effect.
  - `validate(parsed)` runs *before* the parsed value is ever returned as state — corrupt JSON falls back to `initial` without flashing bad data through React.
  - Snapshots are cached per-key so `getSnapshot` returns referentially-stable values (avoids `useSyncExternalStore`'s "infinite loop" warning).
  - Cross-tab sync via `storage` event invalidates the cache and re-emits.
  - Same-tab sync via a manual pub/sub: a setter in tab A notifies any other consumer of the same key in tab A.
- **[FIXED] `syncNodesFromSchema` uses `selectedTableId` from closure.** `components/canvas/schema-canvas.tsx`. The sync now reads `selectedTableIdRef.current` inside its body, so a schema mutation that doesn't change selection stops carrying a stale `selected` flag. `selectedTableId` was removed from the callback's dep array (no longer needed) and a separate small effect kicks the sync when the selection id flips so the visual highlight updates without waiting for the next schema change.
- **[FIXED] Concurrent schema PATCH has no optimistic-concurrency check.** `app/api/schemas/[id]/route.ts` + `lib/workspace-context.tsx`. Wire diagram:
  1. The route now accepts an optional `expectedVersion: number` in the PATCH body. When present it uses `prisma.schema.updateMany({ where: { id, version: expectedVersion }, data })` — a single conditional write that closes the read-then-write window. If 0 rows match, the route returns `409 VERSION_MISMATCH` with `{ currentVersion, updatedAt }` so the client can resync.
  2. `loadSchemaIntoCanvas` snapshots the server-observed `version` into `currentVersionRef`. Every PATCH (autosave + Ctrl+S `saveNow`) now sends `expectedVersion: currentVersionRef.current`. The successful response carries the new `version`, which we re-snapshot for the next call.
  3. On 409 the client toasts "schema was updated in another tab" and calls `loadSchemaIntoCanvas(id)` to realign with the server. The skip-next-autosave guard prevents an immediate re-PATCH that would clobber the freshly-loaded data.
  4. Omitting `expectedVersion` (older clients) preserves the legacy last-write-wins behaviour rather than breaking those sessions.
- **[FIXED] `saveTimerRef` not cleared on unmount.** `lib/workspace-context.tsx`. Provider-unmount `useEffect` cleanup now: clears `saveTimerRef`, sets an `unmountedRef` so any post-mount async branches bail before touching state, and aborts every in-flight PATCH via a per-call `AbortController` registered in `inFlightAbortersRef`. The save handler treats `AbortError` as a silent skip (no error toast, no setState) so dev strict-mode and production route changes both unmount cleanly without firing setState on a torn-down tree or surfacing spurious "Save failed" toasts.
